A Java AWT/Swing Graphics drop-in replacement that renders to the TikZ/PGF (LA)TEX language.

Motivation

People often write simulations in Java, usually including some visualizations.  It is often desirable to use some of these visualizations as figures in papers.  Using screenshots is obviously not desirable since that would produce a raster image when the original was vector.

JTikZ solves this problem by providing a new class, TikzGraphics2D that extends off of java.awt.Graphics2D. This class can be passed to any component's paint() function and will output human readable TikZ/PGF code that can be cut/pasted into a document.

Current Status & Installation

JTikZ currently supports  lines, shapes/polygons, ellipses, colors, alpha, arcs, curves/splines, and text.

You can download and compile the code as follows: 

$ svn co https://jtikz.svn.sourceforge.net/svnroot/jtikz jtikz
$ cd tikz
$ ant jar
$ java -jar lib/example.jar
